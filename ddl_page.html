DDL Commands

Certainly! Here's a question for each DDL command along with a brief explanation and solution:

CREATE

Question: Create a table named `Students` with columns for `student_id`, `first_name`, `last_name`, and `date_of_birth`.

Solution:

CREATE TABLE Students (
    student_id INT PRIMARY KEY,
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    date_of_birth DATE
);

Explanation: The CREATE command is used to define new database objects like tables, indexes, views, etc. In this example, a table named `Students` is created with specified columns. It's essential for setting up the structure of the database.

ALTER

Question: Add a new column named `email` of type VARCHAR(100) to the `Students` table.

Solution:

ALTER TABLE Students ADD COLUMN email VARCHAR(100);

Explanation: The ALTER command is used to modify the structure of existing database objects without dropping and recreating them. In this case, a new column `email` is added to the `Students` table, allowing for additional information to be stored.

DROP

Question: Delete the `Students` table from the database.

Solution:

DROP TABLE Students;

Explanation: The DROP command is used to delete existing database objects like tables, indexes, views, etc. In this example, the `Students` table is removed from the database. It's useful for cleaning up unused or unnecessary objects.

TRUNCATE

Question: Truncate the `Sales` table to remove all records.

Solution:

TRUNCATE TABLE Sales;

Explanation: The TRUNCATE command removes all rows from a table, effectively resetting it to an empty state. It's faster than DELETE because it deallocates data pages in one go, making it ideal for cleaning up large tables without keeping a log of individual row deletions.

RENAME

Question: Rename the `Customers` table to `Clients`.

Solution:

RENAME TABLE Customers TO Clients;

Explanation: The RENAME command is used to change the name of an existing database object, such as a table, view, or column. In this case, the `Customers` table is renamed to `Clients`. It's helpful for reorganizing database structure or providing more meaningful names to objects.
